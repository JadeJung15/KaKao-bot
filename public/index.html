픽셀곰 브릿지

화면 감지 없이 카카오 알림을 받아 오픈채팅방의 자동응답, 출석, 포인트, 채팅랭킹, 방장봇 입장확인 연동을 운영합니다.

도입 절차 테스트 문의
알림 기반 카카오 알림으로 들어온 메시지만 처리합니다.
다중방 준비 방별 설정, 관리자, 입장확인 문구를 분리합니다.
판매형 구조 방 운영자가 직접 쓰는 관리 기능으로 확장합니다.

Product

오픈채팅 운영을 앱과 서버로 연결합니다

픽셀곰 브릿지는 봇폰의 카카오 알림을 서버로 보내고, 서버는 방별 규칙에 따라 필요한 답장과 운영 기록을 처리합니다.

알림 실시간 전송

카카오 알림에서 방, 보낸사람, 메시지를 읽어 서버로 전송합니다.

운영봇 명령어

출석, 포인트, 좋아요, 메시지함, 랭킹을 카카오톡 안에서 처리합니다.

방장봇 입장확인

방장봇 환영문구를 입장 신호로 활용해 첫 채팅과 연결합니다.

닉네임 히스토리

채팅 기반으로 닉네임 후보와 이전 활동 기록을 이어 봅니다.

관리자 권한

방별 관리자만 민감한 설정과 기록 명령어를 사용할 수 있습니다.

원본 알림 진단

카카오 알림 원본을 확인해 방별 파서와 자동응답을 안정화합니다.

Multi Room

판매용은 방마다 설정이 달라야 합니다

한 방 전용 문구가 아니라, 각 오픈채팅방이 자신의 방 이름과 입장확인 문구, 관리자, 명령어 사용 범위를 별도로 갖는 구조로 설계합니다.

방 입장확인 문구 운영 기능
픽셀곰 RPG 놀이터 픽셀곰 입장확인 출석, 포인트, 랭킹
테스트 오픈채팅 입장확인 자동응답, 메시지함
고객 운영방 환영 확인 관리자 명령어

Setup

도입은 네 단계로 끝냅니다

  1. 브릿지 앱 설치 봇폰에 픽셀곰 브릿지를 설치하고 알림 접근 권한을 켭니다.
  2. 채팅방 등록 방 이름, 오픈채팅 링크, 입장확인 문구, 관리자 닉네임을 등록합니다.
  3. 방장봇 문구 설정 방장봇 환영메시지를 방별 입장확인 문구로 맞춥니다.
  4. 명령어 테스트 /브릿지, /상태, /출석, /포인트로 알림과 서버 연결을 확인합니다.

Commands

판매형 운영봇에 필요한 기본 명령어

방별로 켜고 끌 수 있는 명령어를 기준으로 운영자가 바로 테스트할 수 있게 구성합니다.

브릿지

/브릿지 /로컬상태 /최근이벤트 10

운영

/상태 /도움말 /메시지

포인트

/출석 /포인트 /포인트순위

히스토리

/닉이력 닉네임 /입퇴장현황 닉네임 /채팅오늘

Scope

정확한 판매를 위해 가능한 범위를 분명히 합니다

픽셀곰 브릿지는 화면 감지를 기본으로 쓰지 않습니다. 따라서 카카오가 알림으로 주는 메시지와 방장봇 환영문구를 기준으로 안정적인 운영 기능을 제공합니다.

가능

  • 일반 채팅 명령어 자동응답
  • 출석, 포인트, 랭킹
  • 방장봇 입장확인 문구 연동
  • 첫 채팅 기반 히스토리 보정

조건부

  • 입장자 닉네임 즉시 확정
  • 퇴장/닉변 실시간 감지
  • 카카오 알림에 없는 시스템 문구
  • 알림이 꺼진 방의 실시간 처리

Pilot

먼저 내부 테스트방으로 검증합니다

판매 전에는 실제 오픈채팅방에서 알림 원본, 방장봇 입장확인, 명령어 응답, 다중방 설정을 순서대로 검증해야 합니다.
