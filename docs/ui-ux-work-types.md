# Pixgom UI/UX 작업 용어 기준

Pixgom 사이트, 관리자 콘솔, 구매자 콘솔, Android 브릿지 화면 작업을 분류할 때 아래 기준을 사용합니다.

| 표현 | 의미 |
| --- | --- |
| UI 개선 | 화면 디자인, 버튼, 색상, 레이아웃 개선 |
| UX 개선 | 사용 흐름, 이해도, 편의성 개선 |
| UI/UX 리디자인 | 화면과 사용성을 같이 다시 설계 |
| 사용성 개선 | 사용자가 헷갈리지 않게 만드는 작업 |
| 정보 구조 개선 | 메뉴, 설정, 기능 배치를 정리하는 작업 |
| 디자인 시스템 정비 | 버튼, 카드, 색상, 폰트, 간격을 통일 |
| 온보딩 개선 | 처음 설치한 사용자가 쉽게 시작하게 만드는 작업 |
| 접근성 개선 | 글자 크기, 대비, 터치 영역, 가독성 개선 |
| 인터랙션 개선 | 누르면 어떻게 반응하는지, 상태 표시 개선 |
| 마이크로카피 개선 | 버튼명, 안내문, 오류 메시지 문구 개선 |

## 적용 원칙

- 요청에 여러 표현이 섞이면 사용자 흐름과 정보 구조를 먼저 정리한 뒤 화면 디자인을 맞춥니다.
- 구매자 화면은 온보딩, 접근성, 마이크로카피를 우선합니다.
- 관리자 화면은 정보 구조, 사용성, 인터랙션 피드백을 우선합니다.
- 버튼, 카드, 상태 배지, 입력폼은 기존 콘솔 디자인 시스템을 유지하면서 정비합니다.
